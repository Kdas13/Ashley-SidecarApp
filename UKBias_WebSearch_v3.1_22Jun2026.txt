JOB 1 — webSearch.ts
--------------------
SHOPPING_PRICE_KEYWORDS: 20-term list covering price/cost/buy/shop/
available/deliver/bottle etc.

biasQueryForUK(query):
  - If query already contains \buk\b, \bengland\b, \bbritain\b, or
    \bbritish\b → return unchanged (no double-append).
  - If any SHOPPING_PRICE_KEYWORDS term appears → append " UK".
  - Otherwise → return unchanged (news, weather, etc. unaffected).

tavilySearch POST body:
  - query: biasQueryForUK(trimmed) — biased before hitting Tavily.
  - country: "gb" — Tavily country filter, forces UK sources.

Note: the CLASSIFIER_SYSTEM already contained "produce a concise
UK-focused query" — that instruction was already present and is not
duplicated. The new layer operates on the classifier's OUTPUT before
it reaches Tavily, so it catches any query the classifier under-biases.
