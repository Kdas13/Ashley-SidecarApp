WHAT THE MODEL RECEIVES — EXACT STRUCTURE
------------------------------------------
The model receives one string (the prompt). There is no system message,
no negative prompt, no inpainting mask, no style reference image.
The entire framing, distance, action, and kit directives exist only as
plain text inside the single prompt string.

This is significant: gpt-image-1 treats all prompt text as a single
instruction block with no hierarchy. Long complex prompts with many
directives can see later or weaker directives overridden by the model's
prior training distribution. Framing constraints ("15-20 metres back",
"Ashley is a small figure") are abstract spatial concepts that diffusion
models are trained to treat as suggestions, not hard constraints.
Identity anchors (STABLE ASHLEY IDENTITY) typically dominate. The model
gravitates toward close framing of named subjects.

CACHE CHECK (lines 2885–2909)
-------------------------------
Before the OpenAI call, the server checks:
  cacheKey = SHA-256(fullPrompt + "|" + mode).slice(0,12)
  if selfieCache.get(cacheKey) → return cached URL (no API call)
  if selfieInFlight.get(cacheKey) → join existing promise (no duplicate call)

The cache is in-memory plus disk-persisted. On server restart it reloads
from disk. Cache keys are prompt-content-based, so changing the football-pitch
clause produces a new cache key and forces a fresh OpenAI call.
