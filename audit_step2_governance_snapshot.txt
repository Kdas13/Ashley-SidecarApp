PREREQUISITE: imageDefaultsExtra must be correctly saved to the server
-----------------------------------------------------------------------
Prior to commit 8b0824e (fix: add imageDefaultsExtra to wirePatch) and
commit c6ddc13 (fix: add imageDefaultsExtra to WireProfile/ProfileUpdate types),
the imageDefaultsExtra field was silently dropped at THREE points:

  1. useProfile.ts (wirePatch mapping) — field never included in PUT body
  2. WireProfile type in aiClient.ts — field absent from server response type
  3. profileFromWire() in aiClient.ts — field stripped on every server response parse

These three gaps meant every save of activity/timeOfDay/season/shotDistance
was a no-op. The onSuccess callback from update.mutateAsync() wrote back a
profile with imageDefaultsExtra: null, which then overwrote the local draft
via the useEffect at line 274, causing the selection to revert to "Auto".

All three gaps are fixed as of the deployed build (commit 531e653f).
