Q1. DB MIGRATION
════════════════════════════════════════════════════════════

Column to add:
  ALTER TABLE messages ADD COLUMN IF NOT EXISTS tags TEXT[];

Pattern used in this repo:
  drizzle-kit generate is BROKEN in this repo (path bug —
  see memory entry drizzle-kit-path-bug.md). All migrations are
  written as raw SQL and applied manually via executeSql in the
  code_execution sandbox. No migration files exist in the repo;
  there is no drizzle/migrations directory.

Safety assessment — SAFE:
  ADD COLUMN IF NOT EXISTS is a non-destructive DDL operation.
  PostgreSQL adds the column with NULL for all existing rows,
  which matches the nullable default. IF NOT EXISTS makes it
  idempotent. No lock escalation, no data loss, no backfill needed.
  TEXT[] is a native PostgreSQL array type; Drizzle ORM supports
  it via the .array() modifier on a text() column definition.

Two steps required:
  Step 1 — DB: executeSql → ALTER TABLE messages ADD COLUMN IF NOT
    EXISTS tags TEXT[];
  Step 2 — Schema: add to lib/db/src/schema/ashley.ts:
    tags: text("tags").array(),
  Then rebuild the lib: pnpm run typecheck:libs so downstream
  @workspace/db types include the new field.
