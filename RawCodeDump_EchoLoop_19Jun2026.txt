QUESTION: On a normal turn, which server message is supposed to set
responseEndReceivedRef = true on the client, and does the client listener
filter on kind === "main"?

ANSWER:

The message is response_end. The client listener is at lines 575–588 (pasted
in Section 1a above):

  575        case "response_end": {
  576          const kind = (msg["kind"] as string | undefined) ?? "main";
  577          if (kind !== "main") break;   ← DOES filter on kind === "main"
  578          responseEndReceivedRef.current = true;

The filter is correct. The server sends response_end with kind: "main" at both
send sites (D and F above). The listener consumes it correctly.

THE STRUCTURAL PROBLEM VISIBLE IN THE CODE:

response_end is ONLY sent from two places (sites D and F above):
  D: runLLMAndTTSPipeline — gated on `session.currentSpeechId === speechId` at line 652.
