WHAT CHANGED
────────────────────────────────────────────────────────────────────────────────

FILE CHANGED: artifacts/mobile/lib/useVoiceCall.ts (one file, one commit)

1. FOUR NEW REFS (around line 215):
   vadProbeUsableRef  — count of non-null metering frames during probe window
   vadProbeNullRef    — count of null metering frames during probe window
   vadModeRef         — "unknown" | "vad" | "blind" (set by probe timer)
   vadProbeTimerRef   — handle for the 2s probe report timer

2. FRAME COUNTER IN VAD EFFECT (top of useEffect on recorder.metering):
   Runs on every invocation, before the existing `if (db === null) return;`.
   db === null → vadProbeNullRef.current++
   db !== null → vadProbeUsableRef.current++
   Counts regardless of whether vadActiveRef is true (probe runs even during
   the 500ms VAD delay window).

3. PROBE TIMER IN openMic() — after recorder.start() and gen check:
   Resets both counters and vadModeRef = "unknown".
   Clears any previous probe timer (handles rapid mic reopen).
   Captures probeGen = turnGenRef.current (stale-segment guard).
   Starts a 2000ms timer. On fire:
     - If turnGenRef.current !== probeGen → skip (stale segment, mic reopened)
     - usable = vadProbeUsableRef.current
     - nullCount = vadProbeNullRef.current
     - mode = usable > 0 ? "vad" : "blind"
     - vadModeRef.current = mode
     - addLog(`VAD_CAPABILITY mode=${mode} usableFrames=${usable} nullFrames=${nullCount}`)

4. CLEANUP ON UNMOUNT (in the existing cleanup useEffect):
   vadProbeTimerRef cleared alongside the existing vadDelayTimerRef cleanup.

────────────────────────────────────────────────────────────────────────────────
HOW TO READ THE OUTPUT
────────────────────────────────────────────────────────────────────────────────

Two seconds after the mic opens on each turn, the log will contain one line:

  VAD_CAPABILITY mode=vad usableFrames=<N> nullFrames=<M>
    → metering is working. N non-null frames received in 2s.
    → VAD silence gate is operative. autoSubmit is a safety net only.

  VAD_CAPABILITY mode=blind usableFrames=0 nullFrames=<M>
    → metering is null the entire probe window.
    → Every turn boundary is governed by the blind 4s timer (autoSubmit-initial
      or autoSubmit-sound-reset), not genuine silence detection.
    → STT hallucination guard is also bypassed (meteringEverReceivedRef stays
      false → guard always passes through).
    → P0-2 fix becomes mandatory.

  VAD_CAPABILITY mode=blind usableFrames=0 nullFrames=0
    → Metering fired zero times in 2s. The useEffect on recorder.metering
      may not be triggering at all. Deeper investigation needed (expo-audio
      version, audio mode, recorder initialisation).
