FILE: artifacts/api-server/src/app.ts
────────────────────────────────────────────────────────────────────
PURPOSE:
Constructs and exports the Express application object. Applies all
global middleware in order: pino-http request logger, CORS, per-path
JSON body parsers (large: 200 MB for /api/chat/image and
/api/chat/transcribe and /api/state/import; standard: 1 MB for
everything else), URL-encoded body parser, and the combined auth gate.
The auth gate is a single inline RequestHandler that skips rate
limiting and auth for /api/healthz, /api/selfies/*, and
/api/user-images/* (public paths), and otherwise chains
apiRateLimit → requireApiKey (Bearer token, middleware/auth.ts) →
requireDeviceId in that order before delegating to the main router.
The router is mounted at /api. ETag generation is disabled globally
(prevents 304 responses that break React Native selfie poll loops).
trust proxy is set to 1 (single Replit proxy hop) so req.ip reflects
the real client IP for rate limiting.

DEPENDENCIES:
- express (Express, IRouter, RequestHandler)
- cors (cors)
- pino-http (pinoHttp)
- ./routes (main router, default import)
- ./lib/logger (logger)
- ./middleware/auth (requireApiKey — Bearer token check)
- ./middleware/deviceId (requireDeviceId)
- ./middleware/rateLimit (apiRateLimit)

EXPORTS:
- app (Express instance, default export) — consumed by src/index.ts

STATE:
- jsonLarge: express.json({ limit: "200mb" }) — module-level constant
- jsonSmall: express.json({ limit: "1mb" }) — module-level constant
- isPublicApiPath(path): inline function used only within app.ts

CONNECTIONS:
None at module level. The pino-http serialiser sanitises req URL
(strips query strings) and res statusCode before logging.

KNOWN ISSUES:
The req.path check for "large" paths uses exact string equality or
startsWith without normalisation; a request to /api/chat/image/
(trailing slash) would fall through to the 1 MB parser and fail on
large uploads. Observed impact: not yet triggered in production.
The console.log at line 570 in chat.ts (CREATE_TICKET_DEDUPE_CHECK)
was not removed — unrelated to app.ts but worth noting as a cross-file
hygiene item.

FILE: artifacts/api-server/src/index.ts
────────────────────────────────────────────────────────────────────
PURPOSE:
Server entry point. Reads PORT from the environment (throws if
missing or non-numeric), starts the Express HTTP server on that port,
then performs four startup actions: (1) calls
restoreRecoveringSessions() to re-register any voice sessions that
were mid-call when the process last exited; (2) calls
initAudioClips() to pre-generate TTS audio clips for silence
lifecycle messages; (3) scans the messages table for rows with
status='streaming' and flips them to 'interrupted', recovering orphan
SSE bubbles from the previous process; (4) starts a 60-second
setInterval keepalive that pings /api/healthz on the public Replit
domain to prevent workflow hibernation. Separately, a 5-minute
proactive scheduler interval calls tickProactive() (first tick 30 s
after boot). Finally, sets up a raw WebSocket server (ws library,
noServer: true) on the same underlying http.Server. Upgrade events
for /api/voice/call are authenticated (Bearer token from
API_AUTH_KEY, X-Device-Id header validated against DEVICE_ID_RE),
then handed to wss. On wss connection: attempts reclaimSession for
reconnects (within 60-second window); evicts any stale active session
for the same deviceId; creates new session; sends call_connected
frame; starts startSilenceMonitor; calls
VoiceOrchestrationService.startOrchestration. On message: routes
binary frames (audio) to a no-op (lastAudioReceivedAt updated only),
JSON speech_interim to handleSpeechInterim, JSON speech_final to
handleVoiceTurn, JSON call_end to registry.finalise, JSON interrupt
to registry.cancelCurrentTurn. On close: marks session as recovering.

DEPENDENCIES:
- ./app (Express app, default)
- ./lib/logger (logger)
- @workspace/db (db, messagesTable)
- drizzle-orm (eq)
- ./lib/proactiveScheduler (tickProactive)
- node:crypto (timingSafeEqual)
- ws (WebSocketServer — @ts-ignore, no bundled types)
- ./lib/VoiceSessionRegistry (registry.*, restoreRecoveringSessions)
- ./routes/voice-call (handleVoiceTurn, startSilenceMonitor)
- ./lib/VoiceOrchestrationService (startOrchestration, handleReconnect,
  handleSpeechInterim)
- ./lib/AudioClipRegistry (initialize as initAudioClips)

EXPORTS:
None — this is the executable entry point.

STATE:
- wss: WebSocketServer (module-level, noServer: true)
- VOICE_WS_PATH: "/api/voice/call" (constant)
- DEVICE_ID_RE: /^[a-zA-Z0-9_-]{8,128}$/ (constant)
- safeEqualBuf: local function using timingSafeEqual

CONNECTIONS:
- Database: messagesTable (startup orphan recovery, UPDATE status)
- Proactive scheduler: tickProactive() every 5 min
- Public Replit domain: GET /api/healthz every 60 s (keepalive)
- WebSocket: /api/voice/call upgrade path, ws library

KNOWN ISSUES:
Binary audio frames received over the WebSocket update
lastAudioReceivedAt and silenceWarningSent, but the actual forwarding
to Deepgram streaming STT returns immediately at line 302 — the
comment reads "Audio forwarding to Deepgram wired in 1G" but no data
is sent to Deepgram. Voice calls use client-side VAD and push-to-talk
only; deepgramStream.ts is loaded but never used in the live pipeline.
The wss object is typed as `any` due to a missing @types/ws install.
API_AUTH_KEY (used for WebSocket Bearer auth) is a different secret
from API_SECRET (used for HTTP X-API-Key auth); both must be set.

PURPOSE:
Implements the two-lane proactive message scheduler. Exports
tickProactive() (called from index.ts every 5 minutes) and
forceTickForDevice(deviceId) (used by the debug-tick route). On each
tick, loads all profiles and for each with a push token and non-"off"

medical_checkin: no recent-message guard, 6/day aggregate cap) and
Companion lane (memory_nudge, conversation_gap: 60-min recent-message
guard, cadence-based cap: low=1, normal=2, high=4) in that order.
Per-category cooldowns apply to both lanes. Quiet hours (22:00–08:00
in the device's IANA timezone, defaulting to UTC) gate both lanes.

sends to proactive_sends table for deduplication and cap tracking.

DEPENDENCIES:
- @workspace/db (db, ashleyProfileTable, proactiveSendsTable,
  messagesTable)

- forceTickForDevice(deviceId) — consumed by routes/proactive.ts
