NOT INITIATED FROM THIS ENVIRONMENT.

The EAS CLI (eas-cli@14.x installed in artifacts/mobile) hangs before
authentication when run from the Replit container. It reaches the version
check stage and stops — it cannot reach expo.dev from this network context.
This is a Replit sandbox networking constraint, not a code problem.

The server-side changes (VoiceOrchestrationService, VoiceIntentClassifier)
are live on restart. The client-side changes (VAD constants, refs, WS handler)
require a new APK build to reach the device.

TO INITIATE THE BUILD, run from your local machine or any environment with
Expo credentials and network access to expo.dev:

  cd artifacts/mobile
  eas build --platform android --profile preview --non-interactive

Or from the project root:

  pnpm --filter @workspace/mobile exec eas build --platform android --profile preview

The build will produce an APK distributed via the internal Expo channel.
No other code changes are required before the build.
