  Insert a permanent "## Location & timezone" block inside
  buildSystemPrompt() itself, so it applies to ALL paths (chat, voice,
  streaming, vision) with no other file changes.
  One file. No schema change. No migration.

  Where: ashleyCoreSpec.ts, inside buildSystemPrompt() near the top
         of the returned string — after the liveState block.
  Content to inject (verbatim, always on):
    "## Kane's location & timezone (permanent, do not ask)
     Kane is based in Lincoln, UK.
     His timezone is Europe/London (GMT in winter, BST in summer).
     Use this for any price/availability answers, store lookups,
     delivery estimates, or time-of-day references. Never ask him
     where he is or what timezone he is in."

  PRO: One change, covers every path including future ones.
  CON: Hardcoded to Kane — not portable if this ever becomes
       multi-user. Acceptable at current single-user scale.

APPROACH B — Add homeLocation column to profile schema + use it in
              buildSystemPrompt()
  Adds a text("home_location") column to ashley.ts schema.
  Requires a manual migration SQL (drizzle-kit is broken in this repo).
  Then buildSystemPrompt reads profile.homeLocation and injects it.
  Kane's row would be updated via a one-off SQL insert.

  PRO: Proper per-user field, survives multi-user.
  CON: Schema change + migration + manual DB update. More moving parts.

APPROACH C — Fix the voice path gap separately by calling
              buildTimeContext() in VoiceContextAssembler
  Move buildTimeContext() from chat.ts to ashleyCoreSpec.ts (or a
  shared util), then call it in VoiceContextAssembler.assemble()
  using profile.timezone as the fallback when no clientNow/tz header.

  PRO: Fixes the voice timezone gap properly using the stored profile tz.
  CON: Only fixes timezone, not location. Doesn't cover all gaps alone.

RECOMMENDED: Approach A for location (hardcode Lincoln/Europe/London
             into buildSystemPrompt — single-user, clean, immediate).
             Combined with Approach C to fix the voice time context gap
             so Ashley knows the actual time during calls.
