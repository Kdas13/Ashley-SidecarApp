================================================================================
ARCHITECTURE DECISION (resolves Atlas blocker #2 + Coda critical #1)
================================================================================

SERVER AUTHORITY MODEL — one path, no ambiguity:

  1. The conversation pipeline (conversation.ts) scans Ashley's LLM response
     for an [image:] marker
  2. If a marker is found, the pipeline:
       a. Strips the marker from the text Kane will see
       b. Creates a pending job in the job store
       c. Calls startImageGeneration() in the background (fire and forget)
       d. Includes { imageGeneration: { jobId, messageId } } in the
          conversation response JSON
  3. The client receives the conversation response, sees imageGeneration,
     and immediately starts polling the job
  4. There is NO separate POST /generate-image endpoint
  5. The client never creates a job — it only polls

This eliminates the duplicate-job risk entirely.

--- 1A. NEW FILE: artifacts/api-server/src/lib/imageIntent.ts ---

Responsibilities: [image:] marker parsing only. Nothing else.
Maximum 80 lines.

Must contain:

  const IMAGE_MODES = ["PORTRAIT","FULL_BODY","SCENE","SELFIE","OUTFIT"] as const
  type ImageMode = typeof IMAGE_MODES[number]

  function parseImageMarker(text: string): { mode: ImageMode; vibe: string } | null
    - Scans text for first occurrence of [image:MODE|vibe]
    - MODE must be one of the five above (case-insensitive match, stored as uppercase)
    - vibe is everything after the pipe up to the closing ]
    - vibe is trimmed and capped at 400 characters (Coda: add max vibe length)
    - If no valid marker found, returns null

  function stripImageMarker(text: string): string
    - Removes ALL occurrences of [image:...] from text (prevents marker leak)
    - Only applied to Ashley's LLM output — never to user messages, memory
      entries, or retrieved content (Coda: assistant-only marker parsing)

Export both functions. Export IMAGE_MODES and ImageMode.

--- 1B. NEW FILE: artifacts/api-server/src/lib/imageJobs.ts ---

Responsibilities: in-memory job store. Nothing else.

Job type:
  interface ImageJob {
    status: "pending" | "ready" | "failed"
    imageUrl: string | null
    messageId: string
    createdAt: number        // epoch ms
    errorMessage: string | null
  }

Rules:
  - Store: const jobs = new Map<string, ImageJob>()
  - KNOWN LIMITATION: jobs are lost if the server process restarts. A restart
    while generation is in flight will cause the client to receive 404 on poll.
    The client must handle 404 as equivalent to "failed". This is accepted for
    7A and documented here. Do not add database persistence in 7A.

  - pruneExpiredJobs(): removes jobs where:
      (a) status is "ready" or "failed" AND age > 10 minutes, OR
      (b) status is "pending" AND age > 10 minutes
    NOTE: do NOT prune jobs that are "pending" and younger than 10 minutes
    (Coda: pending-job prune edge case)

  - Background prune interval: call pruneExpiredJobs() on a setInterval
    every 5 minutes (Atlas: stale-job cleanup interval separate from poll)
    This interval must be started once at server startup, not per-request.

  - Functions to export:
    createJob(jobId: string, messageId: string): void
    updateJob(jobId: string, update: Partial<ImageJob>): void
    getJob(jobId: string): ImageJob | undefined
    pruneExpiredJobs(): void

--- 2A. POLLING LOGIC (index.tsx) ---

After the conversation response arrives:

  1. Check if response.imageGeneration is present and has a jobId
  2. If yes: call startPolling(jobId, messageId)
  3. startPolling must:

       - Before starting a new poll loop, cancel any existing one

       - On "pending": continue polling
       - On "ready": call stopPolling(), then updateMessageWithImage(messageId, imageUrl)
       - On "failed": call stopPolling(), then updateMessageWithError(messageId)
       - On 60 polls elapsed (3 minutes): call stopPolling(), then
         updateMessageWithError(messageId) — timeout (Atlas: poll timeout)
  4. stopPolling(): clears the interval ref

     (Atlas: cancel poll when user leaves chat)
