BUG 2 — Governance routing not firing
================================================================================

Two changes were made. Both are server-only. No mobile change was needed.

CHANGE 1 of 2
File: artifacts/api-server/src/routes/chat.ts
Function: generateAshleySelfie
Lines before fix: ~2338–2348
Lines after fix: ~2337–2350

What changed:
The governance block was inside an if (governance) guard. When the mobile
client had not yet synced a governance snapshot (null on first run, cold
start, or any run before a profile save), governance was null, the guard
was false, applyGovernance was never called, and imageMode stayed as
whatever the vibe classifier returned — PORTRAIT_MODE for a generic "send
me a pic" request.

  BEFORE:
    let governedVibePrefix = "";
    if (governance) {
      const govResult = applyGovernance(
        governance,
        imageMode,
        clientNow ?? new Date(),
        clientTz ?? "UTC",
      );
      imageMode = govResult.imageMode;
      governedVibePrefix = govResult.vibePrefix;
    }

  AFTER:
    const {
      imageMode: governedImageMode,
      vibePrefix: governedVibePrefix,
    } = applyGovernance(
      governance ?? {},
      imageMode,
      clientNow ?? new Date(),
      clientTz ?? "UTC",
    );
    imageMode = governedImageMode;

Effect: applyGovernance now runs on every selfie request. null governance
is passed as {} (empty object), which triggers all-auto handling. See
Change 2 for what all-auto now means.

CHANGE 2 of 2
File: artifacts/api-server/src/lib/imageGovernance.ts
Function: applyGovernance
Lines before fix: ~210–223
Lines after fix: ~210–237

What changed:
applyGovernance treated "auto" on both camera and composition as "no
preference, keep imageMode as-is." For PORTRAIT_MODE input with both
fields on auto, the function returned PORTRAIT_MODE unchanged. Per the
spec, "auto" means the product default applies, not "no opinion."

Product defaults per the spec document:
  Composition auto → Environment-Centric → SCENE_MODE
  Camera auto      → Wide Room          → SCENE_MODE
  Occupancy auto   → with-kane-and-cats

  BEFORE:
    if (imageMode === "PORTRAIT_MODE") {
      const cameraMapped = cameraPref !== "auto"
        ? CAMERA_TO_IMAGE_MODE[cameraPref] : undefined;
      const compositionMapped = compositionPref !== "auto"
        ? COMPOSITION_TO_IMAGE_MODE[compositionPref] : undefined;
      resolvedImageMode = cameraMapped ?? compositionMapped ?? imageMode;
    }

    const occupancyClause =
      occupancyPref !== "auto" && occupancyPref !== "solo"
        ? buildOccupancyClause(occupancyPref, resolvedEnvKey)
        : "";

  AFTER:
    if (imageMode === "PORTRAIT_MODE") {
      const cameraMapped = cameraPref !== "auto"
        ? CAMERA_TO_IMAGE_MODE[cameraPref] : undefined;
      const compositionMapped = compositionPref !== "auto"
        ? COMPOSITION_TO_IMAGE_MODE[compositionPref] : undefined;
      if (cameraMapped !== undefined || compositionMapped !== undefined) {
        // At least one axis has an explicit preference — honour it.
        resolvedImageMode = cameraMapped ?? compositionMapped ?? imageMode;
      } else {
        // Both "auto" → product default: SCENE_MODE.
        resolvedImageMode = "SCENE_MODE";
      }
    }

    const effectiveOccupancy = occupancyPref !== "auto"
      ? occupancyPref : "with-kane-and-cats";
    const occupancyClause =
      effectiveOccupancy !== "solo"
        ? buildOccupancyClause(effectiveOccupancy, resolvedEnvKey)
        : "";

Priority contract preserved: explicit camera > explicit composition >
product default. If either axis is set explicitly, the explicit value
wins. Product default (SCENE_MODE) only fires when both are "auto."

Combined effect of Changes 1 and 2: every selfie request now goes through
applyGovernance. With no governance params sent (or all-auto), PORTRAIT_MODE
is upgraded to SCENE_MODE. Environment is auto-selected from time/day/season.
Occupancy defaults to with-kane-and-cats.
