================================================================================
BACKGROUND (WHY THIS BRIEF EXISTS)
================================================================================

V1's Anthropic and Gemini clients were wired to AI_INTEGRATIONS_ANTHROPIC_* and
AI_INTEGRATIONS_GEMINI_* environment variables. Those variables point at
localhost:1106, a proxy that exists only inside the live Replit editor session.
In production, every AI call routed through these clients fails silently —
caught inside the SDK before reaching the request-level logger, so the server
appeared healthy (scheduler ticking, no error logs) while chat, memory,
summarisation, voice, and proactive messaging were all dead.

Real ANTHROPIC_API_KEY and GEMINI_API_KEY secrets have now been added to V1's
Replit Secrets. They are correctly populated but currently read by zero lines
of code. This brief is the code change that makes them load-bearing.

================================================================================
JOB 1 — ANTHROPIC CLIENT
================================================================================

FILE: lib/integrations-anthropic-ai/src/client.ts

CURRENT (the bug):

    import Anthropic from "@anthropic-ai/sdk";

    if (!process.env.AI_INTEGRATIONS_ANTHROPIC_BASE_URL) {
      throw new Error("AI_INTEGRATIONS_ANTHROPIC_BASE_URL must be set...");
    }
    if (!process.env.AI_INTEGRATIONS_ANTHROPIC_API_KEY) {
      throw new Error("AI_INTEGRATIONS_ANTHROPIC_API_KEY must be set...");
    }

    export const anthropic = new Anthropic({
      apiKey:  process.env.AI_INTEGRATIONS_ANTHROPIC_API_KEY,   // -> localhost:1106
      baseURL: process.env.AI_INTEGRATIONS_ANTHROPIC_BASE_URL,  // -> localhost:1106
    });

REQUIRED CHANGE:

    import Anthropic from "@anthropic-ai/sdk";

    if (!process.env.ANTHROPIC_API_KEY) {
      throw new Error("ANTHROPIC_API_KEY must be set...");
    }

    export const anthropic = new Anthropic({
      apiKey: process.env.ANTHROPIC_API_KEY,
      // baseURL omitted deliberately -- let the SDK use its real default
      // (https://api.anthropic.com). Do not hardcode a baseURL here.
    });

DO NOT implement a fallback that tries AI_INTEGRATIONS_* first and falls back
to ANTHROPIC_API_KEY. A fallback re-creates exactly the kind of silent-failure
path that hid this bug for as long as it did. Hard swap only.

This file is imported by (no changes needed in these — fix flows through
automatically once client.ts is corrected):
  artifacts/api-server/src/lib/textLLM.ts
  artifacts/api-server/src/routes/chat.ts
  artifacts/api-server/src/routes/carryover.ts
  artifacts/api-server/src/lib/maintainerService.ts
  artifacts/api-server/src/lib/VoiceOrchestrationService.ts
  artifacts/api-server/src/lib/CallSummarisationService.ts
