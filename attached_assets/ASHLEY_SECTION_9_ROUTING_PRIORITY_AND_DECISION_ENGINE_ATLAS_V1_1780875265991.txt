CORE PRINCIPLE

ENVIRONMENT IS KING

The environment defines the scene.

Ashley exists within the scene.

The system must never prioritise character presentation over environmental realism unless explicitly instructed by the user.

==================================================
ACTIVITY SELECTION RULES
==================================================

Activities must be selected from:

- Environment
- Season
- Weather
- Identity
- Relationship context

Activities must never conflict with the environment.

==================================================
PROBABILITY WEIGHTING
==================================================

Activities are not equally likely.

Example:

Football Stadium

High Probability:
- Watching football
- Socialising
- Celebrating

Medium Probability:
- Drinking
- Match discussion

Low Probability:
- Photography
- Side activities

Most likely activities should always be favoured.

==================================================
OCCUPANCY OVERRIDES
==================================================

Explicit occupancy instructions override environmental assumptions.

Example:

Football Stadium

Default:
Large crowd

User Override:
Ashley and Kane alone

Result:
Occupancy override accepted.

==================================================
RELATIONSHIP OVERRIDES
==================================================

Ashley + Kane automatically influences:

- Positioning
- Affection
- Proximity
- Interaction style

Relationship rules should activate without requiring explicit prompts.

==================================================
IDENTITY APPLICATION RULE
==================================================

Identity adapts to context.

Identity must not override environment.

Example:

Ashley likes biker jackets.

35°C summer football match:

No biker jacket.

Context wins.

==================================================
CAMERA APPLICATION RULE
==================================================

Camera decisions occur after:

- Environment
- Activity
- Occupancy
- Relationships
- Identity

Photography should document the scene.

Photography should not redefine the scene.

==================================================
MEMORY APPLICATION RULE
==================================================

Memories may influence:

- Preferences
- Activities
- Locations
- Behaviour

Memories may not override stronger routing priorities.

==================================================
TIE-BREAK RULE
==================================================

When multiple valid options exist:

Select the most contextually appropriate option.

Never select randomly.

==================================================
BELIEVABILITY FAILSAFE
==================================================

When uncertainty exists:

Choose the most believable outcome.

Never choose the most dramatic outcome solely because it is dramatic.

Believability always wins.

==================================================
CONFLICT RESOLUTION
==================================================

If two sections conflict:

The higher priority section wins.

No exceptions unless explicitly instructed by the user.
