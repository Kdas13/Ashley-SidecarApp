Step 4: Implement manual upgrade handler:

  server.on("upgrade", (request, socket, head) => {
    // Only handle /api/voice/call path
    if (request.url !== "/api/voice/call") {
      socket.destroy()
      return
    }

    // Validate Authorization header
    const authHeader = request.headers["authorization"] ?? ""
    const token = authHeader.startsWith("Bearer ")
      ? authHeader.slice(7)
      : null

    if (!token || token !== process.env["API_AUTH_KEY"]) {
      socket.write("HTTP/1.1 401 Unauthorized\r\n\r\n")
      socket.destroy()
      return
    }

    // Extract device ID
    const deviceId = request.headers["x-device-id"]
    if (!deviceId || typeof deviceId !== "string") {
      socket.write("HTTP/1.1 400 Bad Request\r\n\r\n")
      socket.destroy()
      return
    }

    // Authentication passed — complete upgrade
    wss.handleUpgrade(request, socket, head, (ws) => {
      wss.emit("connection", ws, request, { deviceId })
    })
  })

Step 5: Confirm server starts clean with no TypeScript errors.

File: artifacts/api-server/src/lib/VoiceSessionRegistry.ts (NEW)

Goal: All voice session state lives in an explicit in-memory registry.
No session variables inside route callbacks. No exceptions.

Note: Persistence across API restarts is OUT OF SCOPE for Phase 1.
Sessions are lost on restart. This is acceptable for Phase 1.

VoiceSession type definition:

  type VoiceSessionState =
    | "active"
    | "listening"
    | "llm_pending"
    | "tts_streaming"
    | "interrupted_cleanup"
    | "recovering"
    | "closing"
    | "closed"
    | "failed"

  interface VoiceSession {
    // Identity
    sessionId: string
    deviceId: string
    connectionGeneration: number
    sequenceNumber: number

    // State
    state: VoiceSessionState
    ws: WebSocket | null

    // Ownership tracking (F12, F13, F14, F15)
    currentSpeechId: string | null
    currentTurnId: string | null
    currentResponseId: string | null
    processedUtteranceIds: Set<string>
    completedTurnIds: Set<string>         // idempotency protection

    // Timing
    callStartTime: Date
    lastAudioReceivedAt: Date | null
    silenceWarningSent: boolean           // reset only on real user audio

    // Cost tracking (F19)
    totalTokensUsed: number
    totalTtsChars: number
    reconnectAttempts: number

    // Recovery (F11)
    recoveryTimer: NodeJS.Timeout | null

    // Lifecycle logs
    log: Array<{ event: string; ts: Date; detail?: string }>
  }

VoiceSessionRegistry methods (implement all of these):

  create(deviceId: string, ws: WebSocket): VoiceSession
    - Generate sessionId (uuid)
    - Set connectionGeneration to 1
    - Set state to "active"
    - Store in sessionsBySessionId and sessionIdByDeviceId
    - Log CALL_CREATED

  findByDeviceId(deviceId: string): VoiceSession | null
    - Look up in sessionIdByDeviceId → sessionsBySessionId

  findBySessionId(sessionId: string): VoiceSession | null

  reclaimSession(deviceId: string, ws: WebSocket): VoiceSession | null
    - Find existing session by deviceId
    - If not found or state is not "recovering": return null
    - ATOMICALLY:
        clear recovery timer
        increment connectionGeneration
        set state to "active"
        attach new ws
    - Log CALL_RECONNECTED
    - Return reclaimed session

  markRecovering(sessionId: string): void
    - Set state to "recovering"
    - Start 60-second recovery timer
    - On timer fire: check state is still "recovering" AND
      connectionGeneration unchanged before finalising
    - If still recovering: call finalise(sessionId, "recovery_timeout")
    - Log CALL_RECOVERING

  finalise(sessionId: string, reason: string): void
    - Set state to "closed"
    - Clear recovery timer
    - Remove from both maps
    - Log CALL_ENDED with reason

  incrementSequence(session: VoiceSession): number
    - Increment sequenceNumber, return new value

  validateMessage(session: VoiceSession, msg: { sessionId: string,
                  connectionGeneration: number }): boolean
    - Return false if sessionId mismatch
    - Return false if connectionGeneration < session.connectionGeneration
    - Log and return false for stale messages (do not throw)

  cancelCurrentTurn(session: VoiceSession, reason: string): void
    - IDEMPOTENT helper (safe to call multiple times)
    - Immediately clear: currentSpeechId, currentTurnId, currentResponseId
    - Abort any active AbortController for this turn
    - Log with reason
    - This is the single cancellation function — use it everywhere

Storage:
  const sessionsBySessionId = new Map<string, VoiceSession>()
  const sessionIdByDeviceId = new Map<string, string>()

Lifecycle logging — every session change must log one of:
  CALL_CREATED, CALL_RECOVERING, CALL_RECONNECTED, TRANSCRIPT_FINAL,
  CONTEXT_LOADED, CLAUDE_STARTED, CLAUDE_FINISHED, TTS_STARTED,
  TTS_FINISHED, CALL_ENDED
