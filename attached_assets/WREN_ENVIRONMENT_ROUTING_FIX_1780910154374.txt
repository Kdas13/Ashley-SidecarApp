BUG 2 — GOVERNANCE SNAPSHOT NOT REACHING SERVER FROM PROFILE SCREEN
================================================================================

Root cause confirmed:
_governanceParams is populated by useImageGate() React hook.
useImageGate() is only mounted in chat.tsx.
When the user is on the profile screen, useImageGate() is not mounted.
_governanceParams is null.
fetchTestSelfie() omits governanceParams from POST body.
Server receives no governance params and runs autoSelectEnvironment().

Fix:
When fetchTestSelfie() is called from the profile screen, it must
read the current profile selections directly from profile state —
not from _governanceParams.

The profile screen already has all the image defaults in state
(composition mode, environment, time of day, season, activity,
shot distance, camera awareness, occupancy). These values must be
passed directly into the POST body as governanceParams when
fetchTestSelfie() is called from the profile screen.

Do NOT rely on useImageGate() being mounted. The profile screen
must construct its own governance snapshot from its own state
before calling fetchTestSelfie().

Proof required:
After fix, pull the server log for a Generate test image call with
football-pitch selected and show:
- imageEnvironmentDefault: "football-pitch" in the received payload
- football-pitch clause present in the final prompt
- Generated image shows a football pitch context
