DATABASE — migrations/0004_call_summaries.sql
---------------------------------------------

Three tables created (PostgreSQL, IF NOT EXISTS, manual migration):

  call_summaries
    id              TEXT PK DEFAULT gen_random_uuid()::text
    session_id      TEXT NOT NULL
    version         INTEGER NOT NULL DEFAULT 1
    status          TEXT CHECK ('committed', 'pending_review', 'superseded')
    topic           TEXT
    tone            TEXT
    open_items      JSONB DEFAULT '[]'
    summary_text    TEXT NOT NULL
    confidence_score INTEGER CHECK (0..100)
    generated_at    TIMESTAMPTZ DEFAULT NOW()
    committed_at    TIMESTAMPTZ
    created_at      TIMESTAMPTZ DEFAULT NOW()

  call_decisions
    id              TEXT PK DEFAULT gen_random_uuid()::text
    session_id      TEXT NOT NULL
    summary_id      TEXT NOT NULL REFERENCES call_summaries(id) ON DELETE CASCADE
    turn_id         TEXT NOT NULL
    decision_type   TEXT CHECK ('action', 'commitment', 'preference', 'question')
    description     TEXT NOT NULL
    resolved        BOOLEAN DEFAULT false
    created_at      TIMESTAMPTZ DEFAULT NOW()

  summarisation_jobs  (recovery + idempotency queue)
    id              TEXT PK DEFAULT gen_random_uuid()::text
    session_id      TEXT NOT NULL UNIQUE
    status          TEXT CHECK ('pending', 'running', 'complete', 'failed')
    created_at      TIMESTAMPTZ DEFAULT NOW()
    updated_at      TIMESTAMPTZ DEFAULT NOW()

Indexes on session_id and status for each table.


CallSummarisationService.ts
---------------------------

SIGNATURE (adaptation from brief):
  static summariseCall(sessionId: string, deviceId: string, callStartTime: Date): void

  The brief specified (sessionId, turns: ConversationTurn[]) but VoiceSession
  has no turns array — turns go straight to messagesTable from handleVoiceTurn().
  The service fetches turns directly from the DB:

    SELECT * FROM messages
    WHERE device_id = deviceId
      AND source = 'voice_call'
      AND created_at >= callStartTime
    ORDER BY created_at ASC

  DB role "ashley" is normalised to "assistant" for the LLM prompt.

ASYNC BODY (self-invoking IIFE — fire-and-forget):

  Step 1: Insert job row (status: 'pending') into summarisation_jobs.
          onConflictDoUpdate on session_id UNIQUE — safe retry.
          If this write fails, log and return. Nothing else proceeds.

  Step 2: Update job status to 'running'.

  Step 3: Fetch turns from messagesTable. Filter to user/assistant only.
          Guard: if 0 turns found, write placeholder summary and mark complete.

  Step 4: Cap at last 150 turns. Short call bypass: < 4 turns → confidence = 100,
          skip LLM scoring.

  Step 5: Build prompt. Call generateChatText(forceProvider: "anthropic",
          maxTokens: 2048). Anthropic is pinned because structured JSON
          reliability matters; Gemini tendency to drop structure is not
          acceptable here.

  Step 6: parseAndSanitizeJSON() — strip markdown fences, extract JSON bounds,
          strict runtime type validation on every field, safety caps:
            topic: max 150 chars
            tone: max 50 chars
            open_items: max 10 items, each max 256 chars
            decisions: max 20 items, description max 500 chars, < > stripped
            confidence_score: Math.max(0, Math.min(100, score))

  Step 7: writeSummary() inside db.transaction():
            a. Supersede all existing 'committed' and 'pending_review' rows
               for this session_id.
            b. Get max version for session, set nextVersion = max + 1 (or 1).
            c. Insert new call_summaries row (status: 'pending_review').
            d. Insert call_decisions rows FK → new summary id.

  Step 8: Update job status to 'complete'.
          On any catch: update job to 'failed', log error.

ADAPTATION SUMMARY (brief vs implementation):
  callLLM('./llm')           → generateChatText from './textLLM'
                               forceProvider: "anthropic"
  turns: ConversationTurn[]  → fetched from messagesTable by
                               (deviceId, source='voice_call', createdAt >= callStartTime)
  session.turns              → not available on VoiceSession — N/A
  middleware signature        → summariseCall(sessionId, deviceId, callStartTime)
