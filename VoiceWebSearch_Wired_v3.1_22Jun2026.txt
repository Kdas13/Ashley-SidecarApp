WHAT CHANGED
------------
Import:
  maybeRunWebLookup imported from ./webSearch.js at line 31.

Logic (runLLMAndTTSPipeline, after context assembly, before streamChatText):
  maybeRunWebLookup(utterance, ctx.messages, true) runs on every turn.
  - utterance  = the STT transcript for this turn
  - ctx.messages = conversation history already assembled by VoiceContextAssembler
  - builderAware = true (matches profile default; VoiceContextAssembler already
    consumed the profile so a second DB read is avoided)

  If the classifier returns {search:false} — the common case — the function
  returns null immediately. No Tavily call, no delay.

  If the classifier returns {search:true, query:"..."} the Tavily call runs
  (5s hard timeout) and the result block is appended to ctx.systemPrompt
  before streamChatText is called. The block contains voice rules telling
  Ashley to answer first and cite sources naturally.

  All non-success outcomes (empty results, Tavily failure, key not set)
  still inject a status block so Ashley knows the lookup ran but didn't
  deliver — consistent with the text chat behaviour.

PLACEMENT IN PIPELINE
---------------------
Stage 5 (context assembly) → [NEW: web lookup enrichment] → speech_start
already sent to client → Stage 6 (LLM streaming). The lookup runs after
the first-token watchdog is armed, so if the lookup + LLM together exceed
8s the thinking clip fires as normal.

PREREQUISITE FOR LIVE RESULTS
------------------------------
TAVILY_API_KEY must be set as a Replit secret on the API Server artifact.
Without it: maybeRunWebLookup returns an "unavailable" block, Ashley
acknowledges she can't check the web this turn, and the call continues
normally. No crash, no silent failure.
